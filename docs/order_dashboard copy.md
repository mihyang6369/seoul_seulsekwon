# 작업지시서: 서울시 슬세권 지수 측정 대시보드 개발

## 1. 프로젝트 개요
본 프로젝트는 서울시 내의 다양한 편의시설 데이터를 활용하여 '슬세권(슬리퍼를 신고 이동 가능한 거리 내의 편의시설 밀집도)' 지수를 산출하고, 이를 시각화하여 사용자에게 제공하는 대시보드를 구축하는 것을 목표로 합니다.

## 2. 활용 데이터 현황
`seoul_seulsekwon/data/cleaned` 폴더 내의 다음 데이터를 활용합니다:
- **교통**: 지하철역(`metro_station`), 버스정류장(`bus_station`)
- **생활/편의**: 스타벅스(`starbucks`), 서점(`bookstore`), 도서관(`library`), 소상공인 데이터(`sosang`)
- **의료/공공**: 병원(`hospital`), 경찰서(`police`)
- **교육**: 학교(`school`)
- **환경/여가**: 공원(`park`)
- **금융**: 금융기관(`finance`)

## 3. 슬세권 지수 산출 로직 (초안)
각 행정동/법정동별로 다음 가중치를 적용하여 지수를 합산합니다.
- **가중치 예시**:
    - 대중교통(지하철, 버스): 30%
    - 생활 편의(스타벅스, 서점, 소상공인): 30%
    - 교육/공공(학교, 도서관, 경찰서): 20%
    - 의료/금융: 10%
    - 녹지(공원): 10%

## 4. 대시보드 주요 기능 요구사항
### 4.1. 메인 대시보드
- **상단 요약**: 서울시 전체 평균 슬세권 지수 및 최고/최저 지역 표시
- **지역 선택**: 구별, 동별 필터링 기능
- **지도 시각화**: 
    - 서울시 행정동별 슬세권 지수를 단계구분도(Choropleth Map)로 표시
    - 특정 지역 클릭 시 상세 편의시설 마커 표시

### 4.2. 상세 분석 페이지
- **지역 비교**: 두 개 이상의 지역(동)을 선택하여 슬세권 구성 요소 비교 (방사형 차트 활용)
- **랭킹 시스템**: 항목별(예: 카페 많은 순, 교통 편리한 순) 지역 랭킹 TOP 10 제공

## 5. UI/UX 가이드라인
- **디자인 컨셉**: 다크 모드를 기본으로 하되, 골드 또는 네온 블루를 포인트 컬러로 사용하여 프리미엄한 느낌 강조
- **인터랙션**: 마우스 오버 시 데이터 툴팁 표시, 필터 변경 시 부드러운 애니메이션 적용

## 6. 향후 일정
- 1단계: 데이터 전처리 및 지수 산출 로직 확정
- 2단계: 기본 대시보드 레이아웃 구현 (Streamlit 또는 Dash 활용 권장)
- 3단계: 지도 인터랙션 및 고도화 시각화 추가
- 4단계: 최종 정합성 테스트 및 배포
