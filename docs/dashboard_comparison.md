# 슬세권 대시보드 버전 비교 분석 보고서

현재 구현된 `app.py`와 제공된 `참조.py` 코드의 주요 차이점 및 특징을 분석한 결과입니다.

## 1. 구조 및 설계 방향

| 구분 | app.py (현재 구현본) | 참조.py (참조 코드) |
| :--- | :--- | :--- |
| **코드 구조** | **Monolithic**: 데이터 로드, 로직, UI가 한 파일에 통합 | **Modular**: 핵심 로직을 `utils.py`로 분리하여 관리 |
| **설계 의도** | 단일 파일로 실행 가능한 완결성 및 안정성 강조 | 유지보수성과 확장성을 고려한 계층적 설계 |

## 2. 주요 기능 비교

### 📍 위치 설정 방식
- **app.py**: 위도와 경도 숫자를 직접 입력하거나 슬라이더로 조절. 명확한 좌표 제어가 가능함.
- **참조.py**: **주소 검색(Geocoding)** 지원. 사용자가 텍스트 주소를 입력하면 좌표로 변환하여 분석 시작. 지도 클릭 시 해당 위치로 즉시 재분석(`st.rerun`)하는 기능 포함.

### ⚙️ 가중치 및 지수 산출
- **app.py**: 100점 만점을 기준으로 한 정량적 점수 산출 및 S~C 등급 부여. 작업지시서의 공식을 엄격히 준수.
- **참조.py**: 가중치를 배수(0.0~2.0) 형태로 조절. 상세 시설 목록을 데이터프레임 형태로 하단에 표출.

### 🗺️ 지도 및 시각화 인프라
- **app.py**: `st_folium`을 사용하여 인터랙티브 지도 제공. 레이더 차트 중심의 시각화.
- **참조.py**: `st_folium`의 클릭 이벤트를 활용한 위치 업데이트 로직이 구현됨. `st.spinner`를 활용한 UX 처리.

## 3. UI/UX 디자인 특징
- **app.py**: 깔끔한 Blue 테마의 CSS 적용. 직관적인 메트릭 카드 사용.
- **참조.py**: `expander`를 활용한 가중치 설정 및 시설 목록 숨김 처리. 복잡한 설정을 깔끔하게 정리.

## 🎯 향후 고도화 제안 (참조.py 기반)
1. **주소 검색 API 도입**: 현재의 좌표 입력 방식에 더해 `geopy` 등을 이용한 주소 검색 기능 추가.
2. **utils 모듈화**: 코드 복잡도가 높아짐에 따라 `utils.py`로 핵심 계산 로직 분리.
3. **인터랙티브 리런(Rerun)**: 지도 클릭 지점을 즉시 분석 기준점으로 삼는 반응형 로직 강화.
